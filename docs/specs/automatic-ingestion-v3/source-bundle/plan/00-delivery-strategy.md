# Delivery Strategy

## Thin vertical slice first

The first usable slice supports one A-tier Markdown/text snapshot, structural units, exact/lexical retrieval, one question, one Researcher, one Evidence Critic, accepted claims, and one answer replay. It deliberately excludes curiosity branching, reverse questioning, vectors, complex formats, rich graph UI, and canonical publication.

## Expansion rule

Add a capability only when the previous layer has measurable fixtures and failure handling. The order is deterministic source → evidence → claims → answers → curiosity/coverage → convergence/authority → information architecture/documents → dashboard/scale.

## Parallelism

Teams may parallelize within a milestone where dependencies permit. Security/policy and evaluation fixtures begin early and accompany every phase.

## Stage-gate evidence

A demo is not a gate. Required evidence consists of passing deterministic tests, agent fixtures, workflow recovery scenarios, labeled-corpus results, and human review where specified.
