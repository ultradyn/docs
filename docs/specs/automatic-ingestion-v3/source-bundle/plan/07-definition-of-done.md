# Definition of Done

## Leaf task

A task is done only when all deliverables exist, every acceptance criterion passes, all named test surfaces have evidence, documentation/contracts are updated, observability and failure behavior are included, and no unrecorded critical risk remains.

## Work package

All child tasks are done; the package exit gate passes; interfaces are integrated with dependencies; recovery and security impacts are tested; results and versions are recorded.

## Milestone

Every required work package is done, the milestone end-to-end scenario passes from a clean environment, known limitations are explicit, and named reviewers approve promotion.

## Production release

Representative real-corpus evidence, configured quality confidence bounds, zero-cache economics, supported-format and data-policy declarations, DR/deletion drills, operator runbook, and rollback path all pass.

A polished demo, a high model-confidence score, or an empty queue is never sufficient by itself.
