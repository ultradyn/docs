# Sol-specific eval expectations

Run the shared core evals. Sol should outperform on:

- structural divergence among three prototypes;
- originality without incoherence;
- difficult signature interactions;
- full-site visual consistency;
- final cross-axis critique;
- fidelity to complex references without architecture bypass.

Fail Sol-specific evaluation when it spends more tokens while producing cosmetic rather than structural variation, repeatedly reopens settled decisions, or adds unrequested spectacle.
