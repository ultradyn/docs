# Banger mode

A banger is not “more effects.” It is a strong idea executed with enough technical and editorial control that the result feels inevitable.

## Required sequence

1. State the emotional/narrative arc.
2. Choose the signature move.
3. Prove it in a focused prototype before building the whole page.
4. Keep supporting sections quieter than the signature.
5. Use real assets or create an explicit asset plan; do not fill the void with abstract gradient blobs.
6. Build a reduced-motion path that keeps the story and interaction intact.
7. Inspect performance on representative hardware and mobile.

## Banger rubric

- **Distinctive:** recognizable from a cropped screenshot without the logo.
- **Specific:** decisions come from the subject's world.
- **Coherent:** type, color, geometry, imagery, copy, and motion agree.
- **Useful:** the page job and primary action remain obvious.
- **Controlled:** one or two moments dominate; the rest creates rhythm.
- **Technical:** effects are robust, responsive, interruptible, and performant.
- **Complete:** both themes unless an intentional single-theme exception is recorded.

## Common false bangers

- giant gradient headline plus floating cards;
- every section with a different effect;
- WebGL background unrelated to the product;
- scroll hijacking;
- unskippable intro animation;
- cinematic desktop with broken mobile;
- a black page with acid accent that could advertise anything.

Replace spectacle with a subject-specific interaction, demonstration, or narrative device.
