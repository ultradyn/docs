# Domain docs

This is a single-context repository.

Before changing behavior, read root `CONTEXT.md` and ADRs under `docs/adr/` that touch the area. Use the glossary's canonical terms in APIs, tests, issues, and UI copy. If a new project-specific concept is unavoidable, add a tight definition and explicitly rejected synonyms.

Surface conflicts with an ADR rather than silently overriding it. New durable decisions use `docs/adr/NNNN-short-name.md` with context, decision, consequences, and status.
