# Idea Register

The register keeps useful ideas visible without forcing all of them into the baseline.

| Idea | Disposition | Notes / activation rule |
|---|---|---|
| Reverse Questioner | Adopted | Core source-led accounting after forward exploration. |
| Human curiosity injection | Adopted | Available at any graph node; participates in closure. |
| Source–claim–question–answer model | Adopted in v3 | Claims are reusable knowledge between evidence and answers/documents. |
| Split Evidence Critic and Curiosity Planner | Adopted in v3 | Separate contexts/contracts; product may still label the combined experience “Questioner.” |
| Perspective-specific Questioners | Reserve | Activate only after repeated role-specific omissions survive forward, reverse, and human mechanisms. |
| Evidence hubs | Adopted selectively | Reusable claim/evidence packs reduce repeated research. |
| Reverse source-unit questioning | Adopted | Run on meaningful uncovered clusters, not every paragraph blindly. |
| Contradiction laboratory | Adopted | Focused authority/scope/supersession investigation. |
| Source-author repair requests | Retained | Convert unresolved gaps/extraction/authority issues into structured requests to source owners. |
| Incremental re-ingestion | Adopted | Hash diff, claim invalidation, affected obligations, focused PR. |
| Claim graph motif analysis | Experimental | Detect suspicious unsupported hubs, contradiction clusters, or orphan claims. |
| Question graph motif analysis | Experimental | Detect excessive depth, repeated rephrasing, and disconnected demand. |
| Two independent ingestion runs | Experimental | Capture–recapture style estimate of undiscovered facets; expensive. |
| Counter-Questioner | Not baseline | Useful checks are covered by deterministic novelty gate and Evidence Critic. |
| Multi-Researcher retrieval ensemble | Experimental | Compare exact/lexical/authority specialists if recall is inadequate. |
| Semantic vector retrieval | Benchmark-gated | Enable only if representative replay improves materially. |
| External research mode | Deferred | Existing-doc ingestion is closed-source by default. |
| Source witness per claim | Retained | Narrow claim-to-span verifier may improve high-risk citation assurance. |
| Human topic curation | Adopted | Checkbox uncovered-topic tree before supplemental exploration. |
| Prompt-prefix branch packing | Adopted as optimization | Zero-cache correctness and economics remain mandatory. |
| Provider warm-session forking | Optional | Runtime optimization only; durable context manifests reconstruct branches. |
| Graph condensation for writing | Adopted | Stabilize SCCs and synthesize in dependency order. |
| FAQ-only publication | Partial mode | Useful when editorial/canonical documentation gate is not met. |
| Source coverage heat map | Adopted | Distinct from answer evidence sufficiency. |
| Claim canonical-home visualization | Adopted | Supports IA review and duplicate-fact detection. |
| Human navigation benchmark | Adopted | Full documentation promotion gate. |
| Model-family diversity | Risk-based | Required for selected critical decisions where data policy permits. |
| Local-only restricted processing | Adopted policy capability | Required for classified corpora that cannot leave boundary. |
| Agent-that-creates-agents | Deferred | Requires mature fixture generation and review; not ingestion v1. |
| Knowledge export to other databases | Retained | Derived export from Git/claim/question records. |
