# Design modes

Pick one primary mode. The brief always wins; these modes prevent the model from applying landing-page composition to every surface.

## Banger

**Job:** create a memorable brand, launch, portfolio, cultural, entertainment, or product-story experience.

**Character:** expressive and singular. It can be maximal, cinematic, playful, tactile, or technically ambitious, but must have a coherent thesis and remain usable.

**Required:** one signature interaction or composition; strong art direction; content and imagery with specificity; disciplined supporting system; purposeful motion; graceful reduced-motion path.

**Failure:** a pile of gradients, parallax, WebGL, glass, and oversized type with no narrative or product job.

## Refined

**Job:** create a premium, editorial, architectural, professional, or understated brand surface.

**Character:** quiet confidence. Type, spacing, image treatment, proportion, and material detail carry the experience.

**Required:** precise measure and rhythm; controlled palette; exceptional content hierarchy; subtle interaction feedback; visible craft at small scales.

**Failure:** emptiness mistaken for luxury, tiny low-contrast text, or generic serif-on-cream styling.

## Product

**Job:** support repeated action, monitoring, comparison, creation, administration, or decision-making.

**Character:** useful before impressive. Dense when the work demands it, calm where attention is scarce, and predictable across states.

**Required:** task hierarchy; realistic data; keyboard and focus quality; loading/empty/error/success states; scanability; responsive priority; no decorative marketing hero unless the product genuinely needs onboarding.

**Failure:** landing-page sections inside an application, card soup, decorative metrics, or low information density.

## Docs

**Job:** help readers learn, find, understand, copy, navigate, and return to technical or structured knowledge.

**Character:** the most tasteful reading tool possible—not a marketing site wearing a sidebar.

**Required:** information architecture; search; stable deep links; readable measure; precise code blocks; navigation state; mobile contents; versioning where relevant; both themes by default; accessible diagrams/tables.

**Failure:** oversized hero, animated decoration, weak code contrast, deep navigation hidden, or beautiful typography with poor findability.

## Reference

**Job:** implement a supplied screenshot, Figma file, design system, or existing surface faithfully.

**Character:** the reference defines composition. The repository defines implementation architecture.

**Required:** source-of-truth inventory; component/token mapping; breakpoint/state assumptions; visual-delta ledger; consumer-path tests; maintainable responsive translation.

**Failure:** “improving” the reference without permission, literal pixel mapping that breaks responsiveness, or passing screenshot tests through a throwaway demo rather than the requested component.
