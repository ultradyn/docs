Every question is a branch.
Every answer is a merge.

Ultradyn Docs turns unanswered questions into git branches of work: dictated answers are structured, criticised against the asker's declared goals, integrated across the docs, and merged back — leaving the documentation consistent and one commit richer.

question / in-work merged / goal ✔ contradiction / rejected ✘ fresh-context evaluator
state machine · single source of truth

Question lifecycle

stateDiagram-v2
    [*] --> Asked
    Asked --> AnsweredInline: Librarian satisfies all goals
    Asked --> Logged: any goal unsatisfied
    Logged --> Active: queued P1–P5
    Logged --> Deferred: generated, not yet demanded
    Deferred --> Active: demand match / manual promote
    Active --> InAnswer: answerer claims
    InAnswer --> Integrating: critic DONE
    Integrating --> Merged: checks pass + approval
    Merged --> Accepted: asker accepts
    Merged --> Reopened: asker rejects with reason
    Reopened --> Active
    Accepted --> [*]

DONE is decisive: every declared goal is ✔ or explicitly deferred, and zero contradictions remain unresolved. Good enough for the declared goals — never "perfect."

Loop B — Answer

sequenceDiagram
    participant A as Answerer
    participant S as Structurer
    participant C as Critic (fresh per round)
    A->>S: dictation → transcript (verbatim, immutable)
    S->>C: structured draft
    C->>C: per-goal IGC eval + contradiction scan
    C-->>A: findings + spawn-list (deferred children)
    A->>S: follow-up dictation
    Note over A,C: repeat until DONE

Depth gaps never block: the critic spawns tagged child questions (generated, provenance = parent + finding + goal) and the answer ships. Demand promotes them later.

self-modification · fixtures are the contract

Loop D — Agent-Smith

flowchart LR
    REQ[change request] --> AS[Agent-Smith]
    AS --> GEN[agent.md + schema + fixtures]
    GEN --> CI[CI runs fixtures of all touched agents]
    CI -->|pass| PR[PR — same flow as docs]
    CI -->|fail| AS
    PR --> HUM([maintainer merge])

Every agent ships with golden fixtures — known inputs → expected structured outputs — run in CI on any change to that agent. Generated agents don't merge without passing fixtures of their own. Fixtures also catch upstream model drift.

tiers with rules — never weighted arithmetic

Priority rules

P1 contradiction-spawned · asker-rejected reopened
P2 unsatisfied goal on active question · demand-promoted deferred
P3 raw questions, default
P4 generated depth-1 children
P5 depth-2+ · tagged extra-detail

Every assignment carries a one-line rationale and is overridable at a glance. A contradiction isn't "weightier" — it's qualitatively different: the doc base contains at least one wrong claim, so everything touching both sides is suspect until resolved.
