# DESIGN

## Mode

Banger / Refined / Product / Docs / Reference

## Visual thesis

One sentence connecting audience, job, subject, and visual behavior.

## Character

3–5 specific words. Include what each word changes in the interface.

## References

For each source, list the properties being adopted—not a vague URL dump.

## Anti-references

List rejected properties and the positive replacement.

## Information hierarchy

Rank primary, secondary, tertiary content and actions. Include a compact wireframe or region map.

## Signature move

The one memorable, useful element. State why it serves the job and its reduced-motion/mobile form.

## Layout and density

Grid, max widths, gutters, region behavior, alignment, spacing rhythm, density rules, and responsive transformations.

## Typography

Roles, families/fallbacks, scale, weights, line heights, measure, tracking, numeral/code treatment.

## Semantic color and themes

Token roles and independently tuned light/dark values. Record any justified single-theme exception.

## Geometry and material

Radius scale, borders/rules, surfaces, elevation, shadow, opacity, and container policy.

## Imagery and iconography

Sources, crop/aspect rules, illustration/photo treatment, icon family, empty/missing-asset behavior.

## Motion

Purpose, duration bands, easing, orchestration, interruption, reduced-motion behavior.

## Components and tokens

Existing components/tokens to reuse; permitted additions; forbidden parallel abstractions.

## States

Pointer to `STATE-MATRIX.md` and any visual/state rules.

## Accessibility

Semantic landmarks, keyboard model, focus, contrast, motion, target sizes, announcements, alternatives.

## Quality bar

Mode-specific criteria and required screenshot/viewports.

## Non-goals and settled decisions

Decisions implementation must not reopen without new evidence.
