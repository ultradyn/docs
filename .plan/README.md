Question-driven, git-backed, agent-operated documentation system.

Read in order:

1. `01-goals-and-constraints.md` — problems solved, goals with decisive tests, constraints C1–C7, users & friction budgets
2. `02-agent-architecture.md` — agent roster, four loops (workflow patterns), priority rules, lifecycle (mermaid sources)
3. `architecture.html` — visual companion to 02: open in a browser (loads mermaid + fonts from CDN)
4. `03-specification.md` — consolidated implementable spec: repo layout, schemas, flows, app architecture
5. `04-conversation-transcript.md` — verbatim design conversation (this bundle's own raw/ file)
6. `05-self-review.md` — cross-referencing pass: 5 issues fixed in-bundle, 2 deliberate resolutions, 5 open items
7. `06-task-breakdown.md` — hierarchical tasks, M0–M6, constraint-first ordering

Suggested first actions on review: render architecture.html (self-review §8), then read 05 open items before M0.
