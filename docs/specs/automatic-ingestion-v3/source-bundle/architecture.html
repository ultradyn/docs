Role contracts

The product-facing “Questioner” is two independent agents

ResearcherEvidence CriticClaim ExtractorClaim ReviewerCuriosity PlannerReverse Questioner

Evidence Critic: “Do these references completely and honestly support this question?”
Curiosity Planner: “Given the terminal result, what new source-grounded obligations deserve a child question?”

The Evidence Critic cannot create child questions. The Curiosity Planner cannot change the current evidence verdict. Both run in fresh contexts with separate schemas and fixtures.

Implementation plan

Build the measured core first

  1. A-tier source snapshot, structural source units, exact/lexical retrieval.
  2. Question, EvidencePacket, EvidenceVerdict, and replay fixture.
  3. Claim extraction and fresh Claim Review.
  4. Answer composition from accepted claims.
  5. Only then add curiosity branching, reverse coverage, convergence, IA/document generation, the rich dashboard, and scale.
