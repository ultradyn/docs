# Issue tracker: GitHub

Issues and PRDs live in [`ultradyn/docs`](https://github.com/ultradyn/docs/issues).

Use the connected GitHub app for structured issue, pull-request, label, comment, and review data. Use local `git` for branches and commits. Use `gh` only for gaps such as Actions logs or branch operations when it is installed and authenticated.

Pull requests are not a feature-request/triage surface by default. Change requests produced by Ultradyn Docs may be local branches when the current user cannot act on GitHub.

When an engineering skill says to publish an issue, create it in `ultradyn/docs`. A task is not considered blocked merely because `gh` is absent if the GitHub connector or a local Markdown change request can represent it.
